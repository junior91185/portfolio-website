Broadcasting & DoS Mitigation

Technologies: Linux (Ubuntu), Python

This project was created for my cryptography and security course. It shows how a client and server communicate using UDP and how basic DoS protection can be added.

What the Project Does

  • Uses UDP for simple client–server communication.
  • Broadcasts messages from the client to the server.
  • Applies basic DoS mitigation (rate limiting and blocking).

Why I Built This

This lab helped me practice network communication, socket programming, and defensive techniques for handling high traffic.

Example Usage

Set up the Server to listen for Client broadcast

server_RecieveBroadcast.py

import socket

# Server setup
server_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
server_socket.setsockopt(socket.SOL_SOCKET, socket.SO_BROADCAST, 1)    #this socket designed to accept broadcasting
server_socket.bind(("", 5005))                      # Bind to all interfaces on server port 5005 (hypothetical port)

print("Server is listening for broadcasts...")

Set up the Client to broadcasts messages

client_broadcastingServert.py

import socket

# Client setup
client_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM) 
client_socket.setsockopt(socket.SOL_SOCKET, socket.SO_BROADCAST, 1) 
client_socket.settimeout(2)

message = b"Broadcasting message to server"
client_socket.sendto(message, ('broadcast', 5005))  # Send broadcast for all devices listening to port 5005

try:
# Receive response from server
    data, addr = client_socket.recvfrom(1024)
    print(f"Received response: {data.decode()} from {addr}")
except socket.timeout:
    print("No response received.")

Denial-of-Service (DoS) Mitigation in Action

The server safeguards itself against excessive traffic by implementing rate limiting and blacklisting abusive clients.

I used client_broadcasting_loop.py to simulate a DoS attack by sending rapid, repeated messages to the server. To handle this, I built anti_DoS_server_Advanced.py, which detects when a client sends too many requests and temporarily blocks them. If the same client triggers the limit multiple times, the server permanently blacklists them. This shows how basic rate limiting and blocking can help protect a server from overload.
